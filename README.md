# CHOKERZ — амуниция для животных

## Описание проекта

Интернет-магазин «CHOKERZ» на CMS 1С-Битрикс (Бизнес редакция).

**Цель:** Разработка адаптивного интернет-магазина с каталогом товаров, корзиной, личным кабинетом, блогом и интеграцией с маркетплейсами (Озон, ВБ, ЯМ).

## Технический стек

- **CMS:** 1С-Битрикс: Управление сайтом — Бизнес редакция
- **Frontend:** Vanilla JavaScript (ES6+), CSS Flexbox/Grid, BEM-нейминг
- **Шрифты:** Montserrat cyr (основной), дополнительные шрифты для заголовков
- **Изображения:** WebP, SVG, Lazy Load

## Структура проекта

```
/local/
├── components/          # Кастомные компоненты
├── templates/           # Шаблоны сайта
├── php_interface/       # Кастомизация ядра
├── modules/             # Кастомные модули
└── tools/               # Вспомогательные скрипты
```

## Инфоблоки

1. **Каталог товаров** — товары с торговыми предложениями (размер, цвет, материал)
2. **Страницы** — контентные страницы
3. **Блог** — статьи и видео
4. **Отзывы** — отзывы с Озона и ВБ
5. **Видео** — видеоматериалы
6. **Преимущества** — блоки преимуществ

## Функционал

- Адаптивная верстка (Desktop, Tablet, Mobile)
- Каталог с фильтрацией и сортировкой
- Корзина и оформление заказа (sale.order.ajax)
- Личный кабинет с профилем, заказами, избранное (wishlist)
- Поиск по товарам и контенту
- Интеграция с маркетплейсами (кнопки покупки)
- Импорт товаров из Озона (API)
- Синхронизация отзывов (Озон API + ВБ парсинг)
- Интеграция доставки: СДЭК, Почта России, Озон
- Интеграция оплаты: ЮKassa, Robokassa
- Формы: подписка, контакты, опт
- SEO оптимизация (мета-теги, каноникалы, пагинация)
- Производительность (кэширование, WebP, lazy load)

## Установка и настройка

### 1. Установка 1С-Битрикс

1. Скачать дистрибутив 1С-Битрикс Бизнес редакции
2. Загрузить на сервер https://dev.chokerz.ru/
3. Запустить установку через веб-интерфейс
4. Настроить подключение к БД

### 2. Настройка проекта

```bash
# Клонировать репозиторий
git clone <repository-url>
cd chokerz

# Установить зависимости (если есть)
npm install

# Скопировать файлы в /local/
# (все кастомные файлы размещаются в /local/)
```

### 3. Создание инфоблоков

Выполнить настройку инфоблоков через панель администрирования Битрикс.

### 4. Настройка интеграций

- Получить API ключи от маркетплейсов (Озон, ВБ)
- Настроить модули доставки (СДЭК, Почта, Озон)
- Настроить модули оплаты (ЮKassa, Robokassa)

## Разработка

### Требования к коду

- Читаемость и масштабируемость
- Отсутствие хардкода
- Комментирование изменений в `/local/php_interface/` и `/local/components/`
- BEM-нейминг для CSS
- Vanilla JavaScript (ES6+)
- Разработка только в директории `/local/`

### Запрещено

- Изменение файлов в `/bitrix/`
- jQuery и подобные библиотеки
- Кастомные таблицы для стандартных сущностей
- Хардкод данных в шаблонах
- Inline PHP-логика в header/footer
